About this skill

UX Skill for xPM

A custom skill for AI agents and coding assistants that provides specialized knowledge on best practice patterns for enterprise UX for Projectum xPM. Instead of explaining the domain every time, the skill loads automatically and acts as a UX sparring partner.

Design review and UX audit Share a screenshot, describe a screen, paste a link, or activate during coding. The skill identifies usability issues, references UX heuristics, and suggests specific fixes.
UX strategy and design sparring Describe what you're building and who it's for. Get a framed UX strategy — persona, outcome, platform constraints, and biggest risk — before any specifics. It pushes back on designs that harm efficiency or trust.
UX copywriting Get help writing clear, user-centered copy for error messages, configuration labels, help text, tooltips, and empty states — following the three C's: clarity, concision, and appropriate tone.
Refining functionality Describe a problem or capability and the skill iteratively works you through a feature description with user stories and acceptance criteria to refine user-centered functionality.

Tips for getting good results

  • Give it something to react to: a screenshot, a description, a draft, or an error message.
  • Tell it which persona you're designing for. "Review this for a resource manager in X organization with X context" gets a better answer than "review this screen."
  • For user stories, expect a clarifying question before you get a draft — this is intentional.
  • For copywriting, include full context: what the field is for, who sees it, what they need to do.

How to install

Claude Code
  1. Download the zip file from this link.
  2. Unzip the folder enterprise-ux.zip into ~/.claude/skills/. This creates an enterprise-ux/ folder with SKILL.md and a references/ subfolder.
  3. Restart Claude Code — the skill will appear in your skill list automatically.
  4. It activates when you work on xPM UX, or type /enterprise-ux to invoke it directly.
GitHub Copilot
  1. Download the zip file from this link.
  2. Unzip the folder enterprise-ux.zip and extract the contents.
  3. Paste the contents of SKILL.md and the four files in references/ into .github/copilot-instructions.md in your repository.
  4. Alternatively, add the same content under VS Code Settings › GitHub Copilot › Custom Instructions.
  5. Copilot will apply the xPM UX domain knowledge across all chat conversations in that repo.

Configuration guide for Custom Columns
Prompt
"I uploaded a screenshot of the configuration module for enabling custom columns in our solution (custom columns set by admin in Power Apps). Help me improve the visual design and write a text to guide the user in how to use this configuration option."
Skill output: UX Copywriting
UX Strategy for Configuration Help Text
User
Consultant or admin configurator — infrequent use, high consequence. This person is configuring how data appears in the Timeline for all end users. A misconfiguration is disruptive to correct after deployment.
Core insight
The user needs to understand what each setting does before changing it, not discover the effect by trial and error in production.
Copy rule
Lead with what the setting controls. State the consequence of saving inline — especially that changes apply to all users immediately. Don't hide this in a tooltip.

Columns/Fields

This tab controls which columns appear in the Timeline grid and in the task lightbox — the detail panel that opens when a user clicks a task. Changes apply to all users as soon as you save.

Standard columns

Built-in columns cannot be deleted. You can change their display order and whether they appear in the grid, in the lightbox, or both. Click the edit icon to adjust a column's settings.

Custom columns

These are fields your organization has added. You can edit, reorder, or delete them. Deleting a custom column removes it from the Timeline view, but does not delete the underlying data.

To add a new custom column

  1. Click New Custom Column.
  2. Choose the field type and select the source column from your Dataverse table.
  3. Set whether it should appear In grid (always visible in the task row), In lightbox (visible when a task is opened), or both.
  4. Use the Column and Row values to control its position in multi-column lightbox layouts.
  5. Set an Order value to control its position in the grid — lower numbers appear first, left to right.
  6. Click Save. Your changes take effect immediately for all users.
Note: There is no preview mode. Test all configuration changes in a non-production environment before applying them to a live deployment.

Error message for misconfigured custom column
Prompt
"Write the error message that the end user will see when a custom column for a table is not correctly configured."
Skill output: UX Copywriting
Copywriting approach for error message
Audience
End user (project manager or team member) — not the admin who misconfigured the column. They did not cause this and cannot fix it themselves.
Rule
Answer three questions: what happened, why, and what to do next. Never blame the user. Never expose system internals. Confirm the rest of the view is unaffected — this prevents the user from doubting all their data.

Recommended message: when the column name is known

"[Column name]" can't be displayed

This column is missing required configuration and has been hidden from your view. The rest of your data is unaffected.

Contact your system administrator to complete the setup in Configuration > Columns/Fields.

Variant: when multiple columns are affected or the column name is unavailable

One or more columns can't be displayed

Some columns are missing required configuration and have been hidden. The rest of your view is unaffected.

Ask your system administrator to review Configuration > Columns/Fields.

Rationale

Title: Names the specific column so the user knows exactly what's missing — not a vague "something went wrong." Uses plain language, not a technical error code.

"Missing required configuration": Explains the root cause without exposing field names, Dataverse schema, or developer terminology.

"Contact your system administrator": Routes to the right person with a clear next step — not "try again" or "contact support."

Path reference (Configuration > Columns/Fields): Gives the admin an exact location so they can act without guessing.

"The rest of your data is unaffected": Critical. Without this, the user may question the reliability of all data on screen — this one sentence prevents unnecessary escalation and maintains trust.

Program Dashboard UX Audit
Prompt
"Review this dashboard for a Program Manager in a large corporation. How could the UX of this be improved to better support their everyday task flow?"
Skill output: UX Audit
UX Strategy for New Program Dashboard
User
Program Manager in a large organization — daily user tracking initiative health, budget vs. actuals, and resource workload across a complex program with multiple concurrent workstreams. Decisions made from this screen have downstream consequences for delivery and resourcing.
Core insight
A Program Manager scanning this dashboard needs to immediately know what needs my attention today — not reconstruct a view of program health by interpreting five separate charts.
Platform
Power Apps canvas embedded in xPM model-driven app. Fixed viewport; no hover states on touch. All affordances must be visible, not discoverable.
Biggest risk
A PM misses a flagged initiative or overdue action because the dashboard presents every data point at equal visual weight. The anomaly needs to stand out — it doesn't.
Critical
  • H2: Match between system and real world All three top chart panels share the title "Active Initiatives." The actual topic — Health, Stage/Status, Budget vs. Actuals — appears only in the smaller subtitle. A PM scanning quickly cannot build a mental map of what each panel shows. Fix: Promote the descriptive subtitle to the primary heading. "Active Initiatives" adds no information at the panel level.
  • H1: Visibility of system status No "what needs my attention" signal anywhere on the dashboard. The PM must read and interpret all five charts to form any view of program health — at-risk initiatives carry no more visual weight than healthy ones. Fix: Add a summary status row above the charts: flagged initiatives, overdue gate actions, and budget variance at a glance. This is the signal a PM acts on daily.
  • H1: Visibility of system status Every chart shows current state only — no trends. "At risk: 1" means nothing without knowing whether that number is new, recurring, or improving. Budget totals with no variance indicator are unactionable. Fix: Add trend indicators (↑↓ with a period label, or sparklines) to all KPI-type counts and financial totals.
Workflow efficiency
  • The Dashboard tab is positioned last in an 11-tab row — the highest-value view for a daily program review is the hardest to reach. A PM opening a program must scan the full tab bar every time. Fix: Move Dashboard to position 2, or make it the default landing tab when opening a program record.
  • Work by Resource shows "(blank)" as a resource name — a system artifact representing unassigned work. A PM cannot act on this without knowing what it means. Fix: Label as "Unassigned" and visually distinguish it from named resources so it reads as a prompt to assign, not a data error.
  • "5% Complete" in the top header bar is a raw number with no context — on track for this program's current stage, or behind? Fix: Show expected alongside actual (e.g., "5% of 12% expected"), or render as a progress bar with a pace indicator.
  • The Budget vs. Actuals chart legend uses "Gate" and "3. Execute" — internal stage names that are not readable as budget concepts at a glance. Fix: Use plain language: "Approved budget" and "Actuals to date."
Important
  • H7: Flexibility and efficiency of use With 11 tabs, the navigation bar is at capacity. Power users learn the layout; new staff and occasional users scan all tabs every visit. Fix: Review whether all 11 tabs need to be permanently visible — consider grouping low-frequency tabs (Dependencies, Stakeholders) or placing them behind a "More" control.
  • H6: Recognition over recall No "last updated" timestamp on any chart panel. For a dashboard that drives daily decisions, stale data can look current. Fix: Add a discreet "Refreshed [time]" label to each chart panel, or a single timestamp for the full dashboard.
Polish
  • The three icons (refresh, export, expand) on each chart panel are unlabeled. Their function is undiscoverable without clicking. Fix: Add tooltips, or show icon labels on the header row of each panel.
  • The bottom half of the viewport is empty. Fix: Add a sixth panel — a risk or action items list — or allow the PM to configure which panels are shown.
What's working well
  • The chart selection is well-matched to the persona. Health status, stage distribution, budget vs. actuals, and work distribution are all genuine daily signals for a Program Manager — the right data is present, it just needs better prioritization.
  • H6: Recognition over recallThe top header bar (Modified By, Modified On, # of Initiatives) provides clear data provenance. The PM knows immediately how recent the record is.
